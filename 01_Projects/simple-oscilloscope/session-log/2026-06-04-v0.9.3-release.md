## What Happened

The user reported that the V0.9.2 screen behaved like a draggable scrolling plot and that the 1 kHz sine looked too pointy. V0.9.3 fixed the center screen behavior, raised defaults to 20 kSa/s, smoothed sine display, added firmware sine interpolation, built and published release assets, flashed the local board, and verified COM14 returned version `0.9.3`.

## Reusable Lessons

- Scope UI chrome belongs in fixed overlays, not data coordinates.
- After firmware changes, local hardware test requires flashing and protocol verification.
- Packaging fails if the previous packaged exe is running.
