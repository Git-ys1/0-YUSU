# Architecture

## Project Shape

`F:\Project\HyperFrames` is a HyperFrames composition project generated by:

```powershell
npx hyperframes init . --example blank --non-interactive --skip-skills --resolution landscape
```

## Runtime Components

- `productions/001-first-hyperframes-demo/index.html`: root composition and GSAP timeline.
- `productions/001-first-hyperframes-demo/DESIGN.md`: required visual identity input for HyperFrames authoring.
- `productions/002-guobao-fan-lyric-video/index.html`: second root composition with synchronized Chinese lyric timing and original fruit-mecha HTML/CSS visuals.
- `productions/002-guobao-fan-lyric-video/TIMING.md`: VTT-derived lyric/visual timing source of truth.
- `productions/003-tian-binxian-profile/index.html`: third root composition for a 30-second Chinese concept founder profile.
- `productions/003-tian-binxian-profile/assets/research/research-notes.md`: evidence log separating real public industry context from fictionalized character copy.
- `hyperframes.json`: registry and path configuration.
- `scripts/local-hyperframes.mjs`: Node wrapper for stable local CLI execution.
- `scripts/mux-audio.mjs`: Node wrapper for FFmpeg post-render audio mux.
- `scripts/make-energy-bed.mjs`: generated corporate energy tone bed for profile-style narration.
- `package.json`: npm scripts for doctor/lint/inspect/preview/render.
- `productions/001-first-hyperframes-demo/outputs/001-first-hyperframes-demo.mp4`: first rendered artifact with video and audio streams.
- `productions/002-guobao-fan-lyric-video/outputs/002-guobao-fan-lyric-video.mp4`: second rendered artifact with Chinese voiceover and synced lyric visuals.
- `productions/003-tian-binxian-profile/outputs/003-tian-binxian-profile.mp4`: third rendered artifact with Chinese narration and concept-profile disclosure.
- `productions/001-first-hyperframes-demo/snapshots/`: key frame evidence generated by HyperFrames snapshot.
- `productions/003-tian-binxian-profile/snapshots/`: key frame evidence generated by HyperFrames snapshot.

## Dependency Boundary

HyperFrames needs Node.js, Chrome Headless Shell, FFmpeg, and FFprobe. Chrome is managed by `npx hyperframes browser ensure`. FFmpeg/FFprobe are project-local npm dev dependencies so render commands do not depend on the Windows system PATH.

## Data Flow

```text
productions/<id-slug>/index.html + DESIGN.md + assets/
  -> npm run check
  -> HyperFrames compiler + Chrome frame capture
  -> local FFmpeg encode video-only MP4
  -> scripts/mux-audio.mjs when audio is needed
  -> productions/<id-slug>/outputs/<id-slug>.mp4
```
